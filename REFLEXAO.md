O que foi difícil
Minha maior dificuldade inicial foi entender a relação entre o que eu via na minha pasta local e o que o Git "enxergava". No começo, criei as pastas do projeto, mas elas não apareciam no GitHub Desktop nem no terminal como mudanças pendentes. Demorei a entender que, para o Git, uma pasta vazia é como se não existisse; eu precisava de arquivos físicos lá dentro para o rastreamento começar.

Além disso, lidar com o terminal no Windows trouxe barreiras técnicas chatas. Tentar usar comandos de Linux, como o touch, no PowerShell do VS Code resultou em erros que me travaram por alguns minutos. Outro ponto de muita tensão foi o conflito de merge. Ver o código "quebrado" com as marcações de <<<<< HEAD e >>>>> gera uma insegurança sobre qual versão manter e o medo de perder o trabalho feito nas branches paralelas. Sair de uma branch e voltar para a main também foi confuso no início, pois eu esquecia de commitar as mudanças e o Git impedia a troca de ramificação.
